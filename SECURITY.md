# Security Policy

## Supported Versions

Security support is provided for the current release series and the default branch of this repository.

| Version | Supported |
| ------- | --------- |
| `main` | :white_check_mark: |
| `v1.0.x` | :white_check_mark: |
| `< v1.0.0` | :x: |

The `PinCFlow-EMS` submodule is a separate project with its own license and maintenance history. Security issues specific to that submodule should be reported through the `PinCFlow-EMS` project.

## Reporting a Vulnerability

Please do not disclose security vulnerabilities publicly until they have been reviewed.

To report a vulnerability in ENUFFT, contact:

```text
banerjee@iau.uni-frankfurt.de
```

Use a subject line such as:

```text
[ENUFFT security] Vulnerability report
```

Please include:

- A short description of the issue.
- The affected file, function, script, or workflow.
- Steps to reproduce the issue, if possible.
- The expected impact.
- Any suggested fix or mitigation.

I will acknowledge reports as soon as possible and aim to provide an initial assessment within 7 days. If the report is accepted, I will coordinate a fix and release/update the repository as appropriate. If the report is declined, I will explain why.

For ordinary numerical bugs, documentation issues, or reproducibility problems that do not involve security, please open a normal GitHub issue.
