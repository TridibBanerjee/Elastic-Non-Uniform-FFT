##### `progress`

One progress message is written and flushed immediately.

$$
\mathrm{stdout}\leftarrow \mathrm{message}.
$$

##### `print_summary`

The aggregate diagnostics stored in a summary row are printed by the helper

$$
s_c=\mathrm{summarise\_alps\_results}
(\{R_{c,r}:r\in\mathcal R_c\},C_c).
$$

The processed triangle count, median relative RMSE for ENUFFT and CSA over target triangles with reference standard deviation greater than `1 m`, median Parseval ratios over the same non-flat reference set, median ENUFFT $K^\star$, median retained CSA pair count, and median number of window points outside the target triangle are reported.

##### `write_alps_outputs`

For one complete configuration tag $\tau$, four final CSV products are written by the helper.

$$
O_{\Delta}\tau,\qquad O_s\tau,\qquad O_a\tau,\qquad O_m\tau.
$$

Dense spectra and geometry lists are removed from the per-triangle table by `scalar_alps_result_row`. Leading sorted amplitudes from `build_alps_spectra_rows` are stored in the spectra table. Nonzero ENUFFT coefficients and valid CSA coefficients from `build_alps_modes_rows` are stored in the modes table.

##### `write_sweep_summary`

For one mesh-level tag $\tau_S$, the following mesh-level CSV product is written:

$$
O_S\tau_S
$$

The product is built from the ordered list of configuration summaries. No file is written for an empty summary list.

##### `write_pickle_atomic`

A process-specific temporary pickle is written by the helper, after which the target path is atomically replaced.

$$
\mathrm{tmp}=\mathrm{output.suffix}+\mathrm{".tmp."}+\mathrm{pid}.
$$

The reducer is thereby prevented from reading a half-written mesh payload or rank chunk.

##### `distributed_path`

All distributed coordination files are built from one prefix.

$$
p(s)=P_{\mathrm{dist}}+s.
$$

For example, suffix `_Tasks.tsv` becomes `csv/Banerjee_2026_Enufft_Alps_Distributed_<jobid>_Tasks.tsv`.

##### `read_existing_summary`

When a final per-configuration `Summary` CSV already exists, this helper reads its first row so skipped configurations can still be included in the final mesh-level `SweepSummary`.

##### `requested_meshes`

The fixed port uses `mesh_names = ("r2b4", "r2b5")`. The helper normalizes those names, removes duplicates while preserving order, and raises an error if no mesh remains.

##### `build_settings`

Command-line sweep options are no longer parsed by the distributed port. A fixed `SimpleNamespace` is built with

$$
\mathcal G_{\mathrm{mesh}}=(\mathrm{r2b4},\mathrm{r2b5}),\quad
\mathcal W=\text{six window strategies},\quad
\mathcal Q_w=(\mathrm{uniform},\mathrm{voronoi}),\quad
H=(1.0,1.5,2.0),\quad
\Sigma=(1.25,1.5,2.0).
$$

`overwrite_existing` is `False` in the file, so completed configurations with non-empty final summary CSVs are skipped during `prepare` only when their mesh signature and metric version match the current analysis definition.

##### `build_base_case`

For one mesh name, the same case dictionary used by the driver is built by the helper. Preset meshes force their documented spacing and mode defaults through `build_alps_case`. The single-configuration defaults are only a seed before the full sweep expands them.

##### `build_case_records`

The case table is the Cartesian product of requested meshes and sweep settings. A global `config_index`, mesh name, output tag, and case dictionary are stored by each record. For the fixed full sweep there are

$$
2\cdot 6\cdot2\cdot3\cdot3=216
$$

records before existing-summary skips.

##### `summary_path_for_tag`

For one configuration tag $\tau$, this returns the final local-driver-compatible path

$$
\mathrm{Banerjee\_2026\_Enufft\_Alps\_Summary}\tau\mathrm{.csv}.
$$

It is used both to skip completed configurations and to reload skipped summaries during reduction.

##### `worker_dem_payload`

The worker payload keeps only the arrays needed by `process_alps_triangle`, namely flattened DEM coordinates, deplaned heights, mesh triangle ids, triangle means and counts, domain dimensions, map reference coordinates, and smoothing scale.

##### `prepare_mesh_payload`

For mesh $g$, the prepare stage builds the proxy mesh, deplanes the DEM once, selects all triangle ids or the optional first-$N$ subset, and writes

$$
P_g=P_{\mathrm{dist}}\mathrm{\_Mesh\_}g\mathrm{.pkl}.
$$

The manifest records mesh kind, mesh version, nominal spacing, mode count, vertex and triangle counts, nonempty-triangle count, triangle ids, cache path, and sweep-summary tag.

##### `prepare_tasks`

The prepare stage builds the requested cases and triangle ids, then forms

$$
\mathcal Q=
\{(j,c_j,r_j,\tau_j)\}_{j=0}^{N_\tau-1},
$$

where $j$ is the global task id, $c_j$ is the configuration index, $r_j$ is the triangle id, and $\tau_j$ is the output tag. Stale transient distributed files are first removed for the current prefix, including repeated use of the `local` prefix. With the fixed port's `overwrite_existing = False`, an existing non-empty final `Summary` CSV is excluded from $\mathcal Q$ only when its stored `mesh_signature` matches the freshly prepared mesh and its `metric_version` matches the current code. No runtime override is provided in `Run_Alps_Levante.sh`.

##### `load_manifest`

This reads `distributed_path(args, "_Manifest.json")` from the current job prefix.

##### `load_tasks`

This reads `distributed_path(args, "_Tasks.tsv")` and converts `task_id`, `config_index`, and `triangle_id` back to integers.

##### `load_mesh_payload`

Workers open the `cache_file` path recorded for the requested mesh and load the cached deplaned DEM/mesh pickle.

##### `run_worker`

Rank $p$ receives the deterministic stride

$$
\mathcal Q_p=\{(j,c_j,r_j,\tau_j)\in\mathcal Q:\ j\bmod P=p\}.
$$

For each task it initializes the Alps worker globals for the corresponding case and evaluates

$$
R_j=\mathrm{process\_alps\_triangle}(r_j).
$$

The rank writes one atomic `_Rank_######.pkl` chunk file containing all of its rows. No CSV summaries are accumulated in worker completion order.

##### `reduce_chunks`

The reducer reads all rank chunks, groups rows by configuration, checks that every expected triangle id is present, and sorts by `tri_num`. For each complete configuration it computes

$$
s_c=\mathrm{summarise\_alps\_results}(\{R_{c,r}:R_{c,r}\ \mathrm{not\ skipped}\}, C_c),
$$

then calls the same `write_alps_outputs` function used by `Code_Alps.py`. The final CSV products therefore retain the same schema and row ordering as the local driver for the same case.

Skipped configurations from the manifest are reinserted into the mesh-level sweep summary by reading their existing final `Summary` CSV rows.

##### `selected_mode`

Exactly one positional mode is accepted. The accepted mode names are `prepare`, `worker`, and `reduce`. The old flag-style modes are not part of this fixed port.

##### `main`

Three explicit modes named `prepare`, `worker`, and `reduce` are exposed. The selected mode is dispatched by `main`.

#### `Run_Alps_Levante.sh`

Slurm batch-script implementation for the fixed Alps distributed sweep on DKRZ Levante.

##### Index

| Symbol | Shell variable | Value | Purpose |
|---|---|---|---|
| $\mathcal A$ | `--account` | `bb1097` | DKRZ project account |
| $N_{\mathrm{node}}$ | `--nodes` | `32` | requested compute nodes |
| $P_{\mathrm{node}}$ | `--ntasks-per-node` | `128` | Python worker ranks per node |
| $T_{\max}$ | `--time` | `01:00:00` | wall-time limit |
| $P_{\max}$ | `MAX_WORKER_RANKS` | `N_NODES * TASKS_PER_NODE` | upper bound on worker ranks |
| $N_\tau$ | `TOTAL_TASKS` | `_TaskCount.txt` | active triangle task count |
| $P$ | `WORKER_TASKS` | `min(P_max,N_tau)` | ranks passed to `srun` |
| $P_{\mathrm{dist}}$ | `distributed_prefix` | job-prefixed CSV path | shared prefix for manifest, tasks, mesh payloads, and rank chunks |

##### Launcher structure

All threaded numerical libraries are fixed to one thread per rank.

$$
\mathrm{OMP}=\mathrm{MKL}=\mathrm{OPENBLAS}=\mathrm{NUMEXPR}=\mathrm{VECLIB}=1.
$$

Python dependencies are checked and the presence of $A_{\mathrm{DEM}}$ is verified before the `prepare` stage is entered.

The number of active tasks $N_\tau$ determines the actual worker count

$$
P=\min(P_{\max},N_\tau).
$$

If $P>0$, the `worker` stage is started through `srun` and the `reduce` stage follows after worker completion. The `--wait=0` flag keeps faster ranks from causing the whole worker step to terminate while slower ranks are still active. If all requested configurations already have valid summaries, the worker and reducer stages are skipped.

##### `run_case_studies`

At fixed $\{x_q,y_q\}_{q=1}^{Q}$, each terrain case produces

$$
h_q^{(j)}=h^{(j)}(x_q,y_q),
$$

then

$$
\hat h_{m,n}^{\mathrm{dft},(j)},
\qquad
\hat h_{m,n}^{\mathrm{nufft,opt},(j)},
\qquad
\hat h_{m,n}^{\mathrm{nufft,base},(j)},
$$

and finally

$$
e_{m,n}^{\mathrm{opt},(j)}=
\hat h_{m,n}^{\mathrm{nufft,opt},(j)}-\hat h_{m,n}^{\mathrm{dft},(j)},

$$
\{|e_{m,n}^{\mathrm{opt},(j)}|\}_{j,m,n},
\qquad
\{|e_{m,n}^{\mathrm{base},(j)}|\}_{j,m,n}.
$$

##### `build_case`

The function creates the default parameter dictionary and applies one sweep update. The default case contains the physical field constants, ENUFFT parameters, EMS parameters, CSA parameters, and the triangle-domain clipping switch $b_{\Omega T}=\mathrm{True}$. In the sweep, `build_case` is called once for each

$$
p\in\mathcal P,
\qquad
a\in\mathcal A,
$$

so that every algorithm choice is evaluated on a fixed physical mesh and hidden signal.

##### `build_main_triangle`

Given $(L,\eta,\theta_T,\Delta y,u)$, the longest local edge is

$$
\ell_T=\frac{L}{\eta}.
$$

The equilateral height is $\sqrt{3}\ell_T/2$, the plotted height is

$$
h_T=\frac{\sqrt{3}\ell_T}{2}(0.45+0.55u),
$$

and the skew offset is

$$
s_T=0.42(1-u)\ell_T.
$$

The three local vertices are then rotated by $\theta_T$ and shifted to

$$
(\frac{L}{2},\frac{L}{2}+\Delta y\,L).
$$

##### `classify_regions`

For points $p_q=(x_q,y_q)$, triangle vertices $v_i=(x_i,y_i)$, and triangle centre $c=(x_c,y_c)$, the function returns a possible region label $r_q\in\{0,1,2,3\}$. The label `0` is reserved for points inside the triangle.

$$
M_T(q)=\mathrm{points\_in\_triangle\_mask}(p_q,\{v_0,v_1,v_2\}).
$$

The remaining labels are possible labels for points outside the triangle. They are assigned from the direction of each point relative to the same centre. The three vertex directions are

$$
\alpha_i=\mathrm{atan2}(y_i-y_c,x_i-x_c)\bmod 2\pi,
\qquad i=0,1,2,
$$

and the direction from the centre to point $p_q$ is

$$
\theta_q=\mathrm{atan2}(y_q-y_c,x_q-x_c)\bmod 2\pi.
$$

The modulo maps every direction to the same range $[0,2\pi)$, so a direction such as $-\pi/6$ is treated as the equivalent direction $11\pi/6$. The $\alpha_i$ values are the candidate angular boundaries for the outside labels. The $\theta_q$ value is the angle being classified. The code sorts the boundaries.

$$
\alpha_{(0)}\le\alpha_{(1)}\le\alpha_{(2)}.
$$

The interval from $\alpha_{(2)}$ back to $\alpha_{(0)}$ crosses the $2\pi$ to $0$ reset, so the code subtracts $\alpha_{(0)}$ from every direction.

$$
\bar{\theta}_q=(\theta_q-\alpha_{(0)})\bmod 2\pi,
\qquad
\bar{\alpha}_j=(\alpha_{(j)}-\alpha_{(0)})\bmod 2\pi,
\qquad j=0,1,2.
$$

This is only a change of angular origin. It does not change the direction-based label. It makes the first sorted vertex direction equal to zero.

$$
0=\bar{\alpha}_0\le\bar{\alpha}_1\le\bar{\alpha}_2<2\pi,
$$

so the three possible outside labels become ordinary angle intervals.

$$
r_q^{\mathrm{out}}=
\begin{cases}
1, & 0\le\bar{\theta}_q<\bar{\alpha}_1,\\
2, & \bar{\alpha}_1\le\bar{\theta}_q<\bar{\alpha}_2,\\
3, & \bar{\alpha}_2\le\bar{\theta}_q<2\pi.
\end{cases}
$$

The code applies this interval test directly. It starts every outside point at label `1`, then promotes points past each boundary.

$$
r_q^{\mathrm{out}}=1,
\qquad
r_q^{\mathrm{out}}\leftarrow2\ \mathrm{where}\ \bar{\theta}_q\ge\bar{\alpha}_1,
\qquad
r_q^{\mathrm{out}}\leftarrow3\ \mathrm{where}\ \bar{\theta}_q\ge\bar{\alpha}_2.
$$

The returned region keeps the triangle points as `0` and uses the angular label only outside the triangle.

$$
r_q=
\begin{cases}
0, & M_T(q)=1,\\
r_q^{\mathrm{out}}, & M_T(q)=0.
\end{cases}
$$

The labels describe subsets of the supplied point cloud. A label from `1` to `3` can be absent if no supplied point has a direction in that interval.

##### `assign_region_modes`

Four region-specific pairs $(m_r,n_r)$ are sampled from signed nonzero modes. The absolute values are drawn from

$$
m_{\min}\le \lvert m_r\rvert,\lvert n_r\rvert\le N_{\max}^{\mathrm{mode}}-1,
$$

and the signs are chosen independently. When `random_phase=True`,

$$
\phi_r\sim \mathcal U(0,2\pi).
$$

The fixed master seed makes the four hidden modes and phases deterministic for a given case.

##### `build_piecewise_dem`

For point $q$ in region $r(q)$,

$$
h_q=A\cos(\frac{2\pi m_{r(q)}x_q}{L_x}+\frac{2\pi n_{r(q)}y_q}{L_y}+\phi_{r(q)}).
$$

Here $x_q,y_q,L_x,L_y$ use the same horizontal unit, km in the Mono case, so the phase is dimensionless. The amplitude $A$ is a height scale in m, so $h_q$ and the recovered Fourier amplitudes are also in m.

##### `build_mesh_data`

The function combines the triangle geometry, DEM cloud, region labels, region modes, and sampled heights into one dictionary used by all algorithms for a physical setup.

##### `get_analysis_mask`

The ENUFFT support $\mathcal W$ is one of

$$
\mathcal W_T=
\begin{cases}
\Omega_T\cap\Omega, & b_{\Omega T}=\mathrm{True},\\
\Omega_T, & b_{\Omega T}=\mathrm{False},
\end{cases}
\qquad
\mathcal W_S=[0,L]\times[0,L],
\qquad
\mathcal W_C=\{(x,y)\in\Omega:\lVert(x,y)-(\frac{L}{2},\frac{L}{2})\rVert_2\le0.5L\}.
$$

The square-domain mask inside the function is

$$
M_\Omega(q)=
\begin{cases}
1, & 0\le x_q\le L\ \mathrm{and}\ 0\le y_q\le L,\\
0, & \mathrm{otherwise}.
\end{cases}
$$

For the triangle case, `get_analysis_mask` first computes the raw triangle mask $M_T(q)$. When $b_{\Omega T}=\mathrm{True}$, the returned point mask is $M_T(q)M_\Omega(q)$ and the returned area is

$$
\lvert\Omega_T^\Omega\rvert=\mathrm{polygon\_area\_2d}(\mathrm{clip\_polygon\_to\_box}(v_0,v_1,v_2,0,L,0,L)).
$$

When $b_{\Omega T}=\mathrm{False}$, the returned point mask is the raw $M_T(q)$ and the returned area is $\lvert\Omega_T\rvert$. This gives a direct legacy comparison mode while keeping the support decision in one function.

##### `compute_voronoi_weights`

A regular `96 x 96` support grid is built over $\Omega$ and then clipped to any additional support restriction in $\mathcal W$. For the triangle case, the grid membership test uses the raw triangle vertices because the grid itself already lies in $\Omega$. The area factor is $\lvert\Omega_T^\Omega\rvert$ when $b_{\Omega T}=\mathrm{True}$ and $\lvert\Omega_T\rvert$ when $b_{\Omega T}=\mathrm{False}$. Each retained grid cell centre is assigned to its nearest irregular point, so if point $q$ owns $c_q$ grid cells,

$$
\tilde w_q=c_q\frac{\lvert\mathcal W\rvert}{N_{\mathrm{grid}}},
$$

with a small positive floor. The final weights are normalized as

$$
w_q=\tilde w_q\frac{\lvert\mathcal W\rvert}{\sum_r\tilde w_r},
\qquad
\sum_q w_q=\lvert\mathcal W\rvert.
$$

##### `compute_peak_metrics`

For each non-DC conjugate pair, the recovered pair amplitude is

$$
A_{m,n}^{\mathrm{rec}}=\sqrt{\frac{\lvert\hat h_{m,n}\rvert^2+\lvert\hat h_{-m,-n}\rvert^2}{2}}.
$$

The function reports the dominant-mode direction error, amplitude error relative to $A/2$, and largest spurious amplitude. The two amplitude diagnostics are in the same height unit as $A$, which is m for the Mono case.

The hidden triangle signal is reduced to canonical conjugate pairs,

$$
(m,n)\sim(-m,-n),
$$

with the representative chosen so $m>0$ or $m=0,n\ge0$. The direction error is

$$
\Delta\theta=
\min(
\lvert\theta_{\mathrm{true}}-\theta_{\mathrm{peak}}\rvert\bmod 180^\circ,\,
180^\circ-[\lvert\theta_{\mathrm{true}}-\theta_{\mathrm{peak}}\rvert\bmod 180^\circ]
).
$$

##### `compute_csa_reference`

With $b_{\Omega T}=\mathrm{True}$, the CSA FA support is the full square and the SA support is the clipped triangle.

$$
\mathcal W_{\mathrm{FA}}=[0,L]\times[0,L],
\qquad
\mathcal W_{\mathrm{SA}}=\Omega_T\cap[0,L]\times[0,L].
$$

With $b_{\Omega T}=\mathrm{False}$, the SA support is $\Omega_T$.

For Mono, `Code_Mono.py` calls the shared CSA module with `chunk_size=None`, so the FA and SA normal equations are assembled as whole sample clouds. The shared module can use chunking for larger future cases without changing the normal equations being solved. The returned object stores $\hat h_{m,n}^{\mathrm{csa}}$, selected signed modes, selected conjugate-pair count, signed nonzero count, and CSA peak diagnostics once per physical setup.

##### `build_modes_rows`

The sparse spectrum table stores all coefficients with

$$
\lvert\hat h_{m,n}\rvert>10^{-15}
$$

plus the DC coefficient. Each row stores the sweep coordinates, method label, signed mode pair $(m,n)$, real part, imaginary part, absolute value, DC flag, and descending rank by $\lvert\hat h_{m,n}\rvert$.

##### `run_single_configuration`

For one support, weighting rule, and oversampling factor, the function computes

$$
\hat h_{m,n}^{\mathrm{nufft}}=
\mathrm{ENUFFT}(\{x_q,y_q,h_q,w_q\}_{q\in\mathcal W}),
$$

then applies EMS to the raw ENUFFT spectrum to get a sparse conjugate-pair block. It computes ENUFFT peak diagnostics against the hidden triangle mode, attaches the CSA diagnostics already computed for the same physical setup, and returns one detail row, one summary row, and sparse mode rows. CSA mode rows are written only once per physical setup because CSA does not depend on the ENUFFT mask, ENUFFT weighting rule, or ENUFFT oversampling factor.

The stored `triangle_area` column follows the active support. It is $\lvert\Omega_T^\Omega\rvert$ when $b_{\Omega T}=\mathrm{True}$ and $\lvert\Omega_T\rvert$ when $b_{\Omega T}=\mathrm{False}$.

##### `build_parameter_combinations`

The function forms the Cartesian products

$$
\mathcal P=
\{N_{\max}^{\mathrm{mode}}\}
\times\{Q\}
\times\{\eta\}
\times\{\theta_T\}
\times\{\Delta y/L\}
\times\{u\},
$$

and

$$
\mathcal A=
\{\mathcal W\}
\times\{\mathrm{weight}\}
\times\{\sigma\}.
$$

For the current Mono case, $\lvert\mathcal P\rvert=1944$ and $\lvert\mathcal A\rvert=18$.

##### `run_parameter_sweep`

For each $p\in\mathcal P$, the code builds one mesh and one CSA reference. It then loops over all $a\in\mathcal A$ and evaluates the ENUFFT branch. The maximum number of scalar configurations is therefore

$$
\lvert\mathcal P\rvert\lvert\mathcal A\rvert=1944\cdot18=34992.
$$

If a selected ENUFFT support contains fewer than three points, that algorithm case is skipped.

##### `main`

The driver writes

$$
\mathcal T_{\mathrm{detail}},
\qquad
\mathcal T_{\mathrm{mode}},
\qquad
\mathcal T_{\mathrm{summary}},
$$

and the number of valid configurations plus the median ENUFFT and CSA direction errors are printed.

##### `read_last_snapshot`

The function extracts a final two-dimensional field from a stored array. For a field saved as $q(t,z,y,x)$, it returns $q(t_f,z_0,y,x)$. For a field saved as $q(t,y,x)$, it returns $q(t_f,y,x)$, and a stored two-dimensional field is returned unchanged.

##### `read_setup_values`

The function reads the scalar setup values from $\mathcal H$ and falls back to the default manuscript case values when a scalar is absent. The returned dictionary contains the domain lengths, the mountain parameters, and the EMS parameters used by the later CSV rows.

##### `fallback_full_topography`

The analytical mountain uses

$$
R(x,y)=\sqrt{x^2+y^2},
\qquad
E(x,y)=
\begin{cases}
1+\cos(\dfrac{2\pi R}{r_\lambda\lambda_0}), & R\le\dfrac{r_\lambda\lambda_0}{2},\\
0, & \mathrm{otherwise},
\end{cases}
$$

and the unresolved amplitude

$$
a(x,y)=\frac{H_0E(x,y)}{2J(r_h+1)}.
$$

The full displayed height is

$$
h(x,y)=Jr_h\,a(x,y)+\sum_{s=1}^{J}a(x,y)\cos(k_sx+\ell_s y),
$$

with $k_s=2\pi(s-1)/\lambda_0$ and $\ell_s=2\pi/\lambda_0$.

##### `read_topography`

The function reads the saved fine-grid topography when present. If it is absent, it rebuilds the same analytical height on a local grid with $N_x=10r_\lambda(J-1)$ and $N_y=10r_\lambda$, centered on $[-L_x/4,L_x/4]\times[-L_y/4,L_y/4]$.

##### `read_mountain_output`

The function loads the final time $t_f$, grid centers $(x_i,y_j)$, meridional wind $v_{j,i}$, retained launch-mode count $K^\star_{j,i}$, retained launch-power fraction $A_{j,i}$, fine-grid topography $h$, and setup scalars into one dictionary.

##### `build_terrain_rows`

The function writes the terrain table as one row per fine-grid point.

$$
(i,j,x_i,y_j,h_{j,i}).
$$

The stored heights use km because the plotted orography panel is labelled in km.

##### `build_mode_rows`

The function writes one final-time surface row per model column.

$$
(i,j,x_i,y_j,v_{j,i},K^\star_{j,i},A_{j,i},100(1-A_{j,i})).
$$

Inactive launch points have no retained-power fraction and are written with a missing loss value.

##### `build_summary_rows`

The summary row stores the scalar setup, the final time, the number of active launch cells, the range of $K^\star$, the active retained-power range, the maximum active loss $100(1-\min A)$, and the range of $v$.

##### `main`

$\mathcal H$ is read, $\mathcal T_h$, $\mathcal T_K$, and $\mathcal T_s$ are written, and the final time, active-cell count, retained-mode range, and maximum active launch-power loss are printed.
